Recherche par mot dans TOUS les documents et obtenir le contexte du mot dans le texte ainsi que les mots reliés (ski ->
skiing, skis, ski, snowboard, snow, hill):

    GET /adminfile/_search
    {
        "query" : {
            "match" : {"text" : "LeMot"}
        },
        "_source":false,
        "aggregations": {
            "keywords" : {
                "significant_text" : {
                    "field" : "text",
                    "filter_duplicate_text": true,
                    "size": 100
                }
            }
        },
        "highlight": {
            "fields": {
                "text": {}
            }
        }
    }

    Pour voir les mots que ES considère reliés à LeMot:
        -on collapse "hits". Il y aura une liste de mots qui matchent le mieux LeMot. Doc_count est le nombre de
    documents qui ont ce mot qui ont étés trouvés en cherchant pour LeMot; bg_count est le nombre de documents au total
    dans l'index qui contiennent ce mot. Ces mots auront aussi un score assigné par ES.

    Pour voir les emplacements du mot (la phrase le contenant), on re-ouvre hits, et on peut voir:
        -Le score de chaque Document vs le mot
        -Le highlight (l'emplacement) du mot dans le document dans "highlight"

    Notes:
        -si on veut le contenu du document en entier en plus des autres informations, remplacer "_source":false par
    "source":true (juste en-dessous de "query":{...})
        -si on veut plus ou moins que 100 résultats, changer le "size": 100 dans le bloc "significant_text"

Recherche par documents et obtenir tous les termes en ordre d'importance dans le document:

    POST /adminfile/_mtermvectors
    {
        "ids" : ["Path-Report-7316-1775-Redacted"],
        "parameters": {
            "fields": [
                    "text"
            ],
            "term_statistics": true,
            "filter" : {
          "max_num_terms" : 10,
          "min_term_freq" : 1,
          "min_doc_freq" : 1
        }
        }
    }

    Les termes sont dans "terms".
